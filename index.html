Service structure

한눈체크 기능 구성

이번 버전은 실제 조회 로직 없이 한눈체크의 화면 구조와 문구를 먼저 구성했습니다. 추후 API 연동 시 아래 카드들이 실제 결과 영역으로 확장됩니다.

01

사업자등록 상태 확인

사업자번호를 기준으로 계속사업자, 휴업자, 폐업자 여부를 보여주는 결과 영역입니다.

02

통신판매업 정보 확인

온라인 판매 사업자의 신고 여부, 신고번호, 신고일자, 업소상태를 정리하는 영역입니다.

03

쇼핑몰 정보 비교

사용자가 본 쇼핑몰 하단 정보와 공식 등록정보가 같은지 비교할 수 있게 안내합니다.

Safe position

한눈체크는 사기 판정 서비스가 아닙니다

한눈체크는 계좌번호, 전화번호, 사용자 신고 이력, 피해사례 DB를 다루지 않는 방향으로 설계합니다. 서비스의 핵심은 공식 등록정보 확인과 사용자가 직접 비교할 수 있는 체크리스트 제공입니다.

Notice

거래 안전을 보장하지 않습니다

사업자등록 상태나 통신판매업 정보가 정상으로 확인되어도 실제 거래 안전성을 보장하지 않습니다. 결제수단, 환불정책, 배송정책, 후기, 사이트 하단 정보를 함께 확인해야 합니다.

How to use

사용 흐름

조회 기능이 연결되면 사용자는 아래 흐름으로 사업자 정보를 확인하게 됩니다.

1

사업자등록번호, 상호명, 대표자명, 통신판매업 신고번호를 먼저 확인합니다.

2

공식 정보 조회

사업자등록 상태와 통신판매업 신고 정보를 한 화면에서 확인합니다.

3

입력정보 비교

조회된 정보와 쇼핑몰에 표시된 정보가 일치하는지 직접 비교합니다.

Checklist

  • 사이트 하단의 사업자정보와 조회 결과를 비교하세요.
  • 무통장입금만 요구하는지 확인하세요.
  • 가격이 지나치게 낮은 경우 추가 확인이 필요합니다.
  • 환불정책, 배송정책, 고객센터 정보를 함께 확인하세요.

FAQ

자주 묻는 질문

사업자 정보가 정상이면 안전한 쇼핑몰인가요?

아닙니다. 정상 등록정보는 참고 자료일 뿐이며, 실제 거래 안전성을 보장하지 않습니다.

한눈체크에서 사기 여부를 알려주나요?

사기 여부를 판정하지 않습니다. 공식 등록정보 확인과 비교 체크리스트 제공에 집중합니다.

계좌번호나 전화번호도 조회하나요?

조회하지 않습니다. 개인정보와 분쟁 리스크를 줄이기 위해 사업자 정보 확인 중심으로 운영합니다.

입력한 사업자번호를 저장하나요?

서비스 구현 시에도 원칙적으로 조회 입력값을 저장하지 않는 구조를 우선 적용합니다.
