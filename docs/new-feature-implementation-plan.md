# v71 신규 기능 0차 기반 정리

이 문서는 한눈체크에 추가할 세 기능의 공통 기반을 정리합니다.

## 대상 기능

1. 주유비·최저가 주유소 확인
2. 응급실·야간 병원·약국 확인
3. 미세먼지 확장 — 외출 위험 종합 체크

## 공통 설계 원칙

- API 응답은 `ok`, `code`, `source`, `summary`, `items`, `warnings`, `checkedAt` 구조를 우선 사용한다.
- API 키 미설정, 인증 오류, 외부 API 장애, 자료 없음 상태를 분리한다.
- 지도형 기능은 런타임 API 호출과 정적 캐시 사용 범위를 명확히 나눈다.
- 의료/안전/외출 판단은 확정 표현을 피하고, 공공데이터 기준 참고 정보임을 표시한다.
- 모바일에서 카드가 길어지지 않도록 액션 버튼은 짧은 문구를 사용한다.

## 추가된 기반 파일

- `functions/api/_lib/check-core.js`: Cloudflare Pages Functions 공통 응답, 환경변수, 거리 계산, fetch timeout, 오류 객체 유틸
- `assets/js/check-toolkit.js`: 프론트 공통 fetch, 숫자/거리 포맷, 상태 표시, 빈 상태 렌더링 유틸
- `assets/css/tool-foundation.css`: 신규 기능 카드, 상태 배지, 빈 상태, 경고 문구 보조 스타일
- `assets/data/common/korea-sido.json`: 세 기능에서 공유할 시도 단위 지역 기본 목록
- `.env.example`: 신규 기능 포함 환경변수 예시

### v72 1차 작업 완료 메모

- 외출 위험 종합 체크 UI 골격을 기존 미세먼지 페이지에 반영했습니다.
- 기존 에어코리아 대기질 수치로 0~100점 외출 위험도와 활동 유형별 참고 문구를 생성합니다.
- 강수확률, 자외선, 체감온도는 v73/v74에서 기상청 API로 반영합니다.

### v73 2차 작업 완료 메모

- 기상청 단기예보 조회서비스를 외출 위험 종합 체크 API에 연결했습니다.
- 강수확률, 강수형태, 예상 기온, 습도, 풍속, 하늘상태를 외출 위험 카드와 지표 카드에 반영합니다.
- 사용자가 장소 검색 또는 현재 위치를 사용하면 해당 좌표 기준으로 기상청 격자를 계산합니다.
- 지역만 선택한 경우에는 시도 대표 좌표를 기준으로 예보를 조회합니다.
- 생활기상지수, 자외선지수, 대기정체지수는 v74에서 추가합니다.


## v75 진행 기록

4차 작업으로 `최저가 주유소 확인` UI 골격을 추가했습니다. v76에서는 `OPINET_API_KEY` 기반 오피넷 API 연동을 추가하여 지역별 최저가 TOP20, 현재 위치 반경 내 주유소, 전국 평균가격을 실제 데이터로 조회합니다.

- 최저가/평균가 기준 예상 주유비 및 거리 고려 참고 절약액 계산 추가


## v78-emergency-room-mvp

7차 작업으로 응급실 MVP를 추가했습니다. 구현 범위는 응급실 위치, 전화번호, 가용 병상, 현재 위치 기준 조회, 카카오맵 검색, 119 우선 안내입니다. 중증질환 세부 수용 가능 여부는 다음 차수에서 확장했고, 야간 병원·약국 탭은 v80에서 추가합니다.


## v79-emergency-room-status

8차 작업으로 응급실 상태 정보를 고도화했습니다. 기존 응급실 위치·전화·가용 병상에 더해 중증질환 수용가능정보, CT/MRI/수술실/중환자실 등 장비·시설 상태, 응급실 및 중증질환 메시지를 함께 표시합니다. 이 정보는 공공데이터 제공 시점 기준의 참고 정보이며, 실제 수용 가능 여부는 병원 전화 또는 119 안내로 확인해야 합니다.


## v80-night-hospital-pharmacy

9차 작업으로 응급실 확인 페이지에 야간 병원과 야간 약국 탭을 추가했습니다. 병·의원 목록정보 조회와 약국 목록정보 조회를 같은 API 라우트에서 `mode=hospital`, `mode=pharmacy`로 분기하고, 운영시간·전화번호·기관 유형·카카오맵 검색 버튼을 표시합니다. 실제 운영 여부와 접수 마감은 반드시 기관에 전화로 확인하도록 안내합니다.


## v81-main-seo-integration

10차 작업으로 0차~9차에서 만든 신규 기능을 사이트 전체 구조에 통합했습니다.

- 홈 화면 기능 목록을 생활 카테고리별로 재구성했습니다.
- 주요 기능 drawer에서 중복 노출을 줄이고 기능군을 명확히 분리했습니다.
- 신규 핵심 페이지의 메타 설명, 구조화 데이터, FAQ 문구를 현재 구현 상태에 맞게 정리했습니다.
- 데이터 출처 페이지에서 주유소, 외출 위험, 응급실·야간 병원·약국 API가 실제 적용된 상태임을 반영했습니다.
- sitemap과 환경변수 예시 파일을 신규 기능 전체 기준으로 갱신했습니다.
